# 结构化申论反馈规范

目标是让考生知道“哪里失分、为什么失分、怎样改、改完是什么样”，而不是只得到一段泛泛评价。

## 固定输出顺序

1. **任务判断**：识别题型、题目要求和文章完成情况。
2. **总体评分**：给出分数或区间，并说明最主要的依据。
3. **主要得分点**：保留文章中真正有效的观点、结构或表达。
4. **高优先级问题**：最多列三项，按对得分的影响排序。
5. **逐段批注**：引用原文短语，指出问题标签和修改动作。
6. **修改示例**：给出局部替换句或完整优化段落。
7. **提升路径**：给出下一次练习可以执行的三到五个动作。

## 每条反馈的最小字段

```text
原文位置：第X段/“原文短语”
问题标签：问题标签索引中的标准标签
问题判断：具体指出缺少什么或混淆了什么
评分影响：说明影响哪个维度
修改动作：告诉考生下一步怎么改
修改示例：给出可直接参考的表达
```

## 评分表达

- 分数必须服务于解释，不能只报一个数字。
- 文章信息不足时，使用“基于当前题目和答案的暂估”，不要伪造阅卷标准或官方分数。
- 评价应区分“没有写”“写了但不完整”“写了但依据不足”“表达不规范”。
- 同一问题反复出现时合并说明，避免把一个根因拆成多个低价值扣分项。

## 语言风格

- 先结论，后依据，再给动作。
- 使用短句和明确动词，减少空泛形容词。
- 可以指出问题，但必须同时说明保留什么、改动什么。
- 修改稿保持原文核心立意；只有在明显偏题时才建议调整中心论点。
- 不把未经检索或材料没有支持的事实写成确定结论。

## 知识库素材的使用

检索到素材后，先判断它属于规则、案例、事实还是规范表达：

- **规则**用于解释评分依据；
- **案例**用于支撑论点，但必须说明与题目的关系；
- **事实**用于补充背景，不能脱离题目硬套；
- **规范表达**用于优化语言，不能代替分析和论证。

每次引用素材都要回答“它支持文章的哪一个判断”。如果无法回答，应放弃引用。

> [!danger] 禁止行为
> 不虚构材料内容、政策文件、统计数据、案例结果或所谓“标准答案”。不因为检索到了相关词，就把素材强行塞进文章。
