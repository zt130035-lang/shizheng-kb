# 大作文问题诊断矩阵

这份规则用于把“文章写得不好”拆成可定位、可解释、可修改的问题。批改时优先处理影响得分最大的两到三项，不平均罗列小问题。

## 诊断顺序

1. **任务完成**：是否回应题目对象、主题、角度和文体要求。
2. **中心论点**：能否用一句话说清文章主张，且与题目有直接关系。
3. **分论点关系**：分论点是否共同支撑中心论点，彼此是否重复。
4. **段落论证**：每个分论点是否包含判断、解释、依据或案例、落地路径。
5. **素材使用**：案例是否真实可核、与论点相关，是否说明案例带来的启示。
6. **语言规范**：是否准确、简洁、正式，是否存在口语化和空泛套话。
7. **格式规范**：标题、字数、段落、标点和结尾是否符合要求。

## 现象到问题的映射

| 文章现象 | 优先标签 | 批改依据 | 修改动作 |
| --- | --- | --- | --- |
| 开头只重复题目，没有中心判断 | `审题/任务缺失`、`立意/观点模糊` | 读者无法判断文章主张 | 用“对象 + 核心矛盾 + 价值方向”写出一句中心论点 |
| 分论点都是“加强、推进、完善” | `逻辑/论证不足`、`对策/不具体` | 只有态度，没有问题机制或执行路径 | 补充执行主体、具体动作、作用对象和预期结果 |
| 多段都在写同一个意思 | `结构/分论点重复` | 段落不能形成有效分工 | 按问题、原因、影响、对策或不同主体重新分组 |
| 案例写完没有回扣观点 | `论证/案例无分析` | 事实不能自动证明论点 | 在案例后补“这说明……因此……”的机制解释 |
| 观点与材料无对应内容 | `内容/材料脱离` | 申论答案不能只依赖一般常识 | 引用材料中的对象、矛盾或限制条件，再提出判断 |
| 对策只有价值口号 | `对策/缺少路径` | “提高认识”不能直接执行 | 写清谁来做、做什么、怎么做、解决什么问题 |
| 使用绝对化或未经依据的政策判断 | `语言/表述失真` | 可能造成事实和政策风险 | 改为有边界的判断，并说明依据或适用范围 |
| 句子很长但层次不清 | `语言/句式混乱` | 影响阅卷理解和要点识别 | 一句只承担一个主要判断，拆分因果和转折关系 |

## 段落合格结构

大作文分论点段落至少应回答以下问题：

1. 本段要证明什么？
2. 为什么成立？
3. 有什么材料、案例或现实表现可以支撑？
4. 对治理或实践有什么启示？

推荐结构：

> 分论点判断 → 解释原因或机制 → 材料/案例依据 → 具体做法 → 回扣中心论点

## 批改输出模板

对每一个高优先级问题，必须按以下顺序反馈：

1. **原文位置**：引用原文短语或指出第几段。
2. **问题标签**：使用问题标签索引中的标准名称。
3. **失分原因**：说明它如何影响审题、内容、结构、论证或语言得分。
4. **修改动作**：给出可以直接执行的改法。
5. **局部示例**：提供一到两句修改后的表达。

> [!warning] 使用边界
> 不能仅凭语言华丽程度判断高低分。先确认任务完成、观点和论证，再评价表达质量。没有题目和材料时，应明确说明判断依据不足。
