# 申论批改知识库

这是供 Obsidian 维护的申论批改知识库，服务于 `shizheng_kb` 项目的申论批改、润色和范文对比功能。

## 使用方式

1. 在 Obsidian 中打开本目录作为 Vault。
2. 先维护 `评分标准`、`批改风格` 和 `问题标签`。
3. 每次人工批改后，用 `模板/批改案例模板.md` 新建案例。
4. 只有审核通过的内容才设置 `status: published`，供后端导入。

## 目录

- [[首页]]：知识库入口和维护流程
- [[评分标准/总则]]：通用评分原则
- [[评分标准/大作文]]：大作文评分规则
- [[评分标准/小题-概括归纳]]：小题评分规则示例
- [[批改风格/默认批改风格]]：模型输出风格
- [[问题标签/问题标签索引]]：统一问题分类
- [[模板/批改案例模板]]：人工批改案例模板
- [[模板/评分规则模板]]：评分规则模板
- [[模板/素材模板]]：时政和作文素材模板
- [[原始数据索引]]：原项目 data 目录的直接入口

## 发布规则

- `draft`：草稿，不进入线上知识库
- `reviewed`：已人工复核，可以继续完善
- `published`：允许后端导入和检索
- `deprecated`：废弃，不再参与检索

## 后端接入

申论专用向量库使用 `data/essay_knowledge_db`，与原有 `data/knowledge_db` 分开保存。

它会读取：

- `data/topics`
- `data/daily_reports`
- 本 Vault 中 `评分标准`、`批改风格`、`批改案例`、`素材` 和 `问题标签` 下状态为 `reviewed` 或 `published` 的笔记

重建接口：

```text
/api/rebuild-essay-kb
```

原始数据不会因为重建申论库而被修改或删除。

> [!warning] 内容质量
> 申论批改案例必须保留题目、材料、原答案、问题依据、修改后答案和修改理由。只有一段范文或一句评价，不能作为高质量风格样本。
