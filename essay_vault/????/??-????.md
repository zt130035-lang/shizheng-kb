# 小题题型路由规则

先根据题目动词和作答对象识别题型，再决定提取哪些信息。不能用大作文的评价方式批改小题。

## 题型识别

| 题目关键词 | 主要任务 | 必须检查的内容 |
| --- | --- | --- |
| 概括、归纳、概述、列举 | 从材料中提炼信息点 | 对象准确、要点完整、分类清楚、表述规范 |
| 分析、谈认识、谈理解、评价 | 解释现象并形成判断 | 观点、原因、影响、依据、结论之间有逻辑 |
| 提出建议、提出对策、解决问题 | 针对问题设计行动方案 | 主体、对象、动作、路径、保障和结果 |
| 贯彻、落实、宣传、倡议、讲话、发言 | 完成应用文表达任务 | 身份、对象、目的、格式、内容、语气 |
| 介绍、说明、推荐、推介 | 按对象需求组织信息 | 信息筛选、顺序、重点、表达场景 |

## 概括归纳题

- 先确定概括对象，再从材料中找同类信息。
- 现象、原因、影响、对策不能无标记混写。
- 每个要点应尽量使用“概括词 + 具体内容”的结构。
- 检查是否存在同义重复，以及是否遗漏主体、对象或限制条件。

## 综合分析题

推荐检查顺序：

1. 这句话或现象的基本含义是什么？
2. 材料提供了哪些事实或背景？
3. 为什么会出现，产生什么影响？
4. 应当如何理解或解决？
5. 结论是否回扣题目对象？

详细标准：[[评分标准/小题-综合分析]]。

## 提出对策题

每条对策至少包含两个要素，重点对策应尽量覆盖四个要素：

- **主体**：谁来执行。
- **动作**：具体做什么。
- **路径**：通过什么机制、流程或资源完成。
- **结果**：解决什么问题或达到什么效果。

“加强宣传、完善制度、提升能力”只能作为方向，不能直接视为完整对策。

详细标准：[[评分标准/小题-提出对策]]。

## 应用文题

先锁定身份、受众和使用场景，再检查格式。正文要服务于沟通目的，不要把应用文写成泛泛的大作文。批改时分别指出：

- 格式是否完整；
- 称谓和语气是否匹配受众；
- 内容是否覆盖题目要求；
- 号召、说明或汇报是否具体；
- 结尾是否符合文种功能。

详细标准：[[评分标准/小题-应用文]]。

> [!warning] 证据要求
> 小题评分必须尽量回到材料。若材料未提供事实，不应把模型常识补写成“材料要点”；可以标记为“合理延伸”，但不能冒充得分点。

## 申论100题补充路由

综合题中的词句解释优先按“总体释义—展开解释—补充对策”检查；单一题要区分直接摘抄、归纳概括和由对策反推问题三类要点来源。完整规则见[[评分标准/申论100题-方法增补]]。
