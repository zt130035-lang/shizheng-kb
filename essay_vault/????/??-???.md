# 应用文题评分标准

## 核心任务

申论应用文首先是特定身份在特定场景下完成沟通，不是机关正式发文的版式考试。批改顺序应为身份和任务完成度优先，格式与语体随后。

## 评分维度

| 维度 | 检查重点 |
| --- | --- |
| 身份与对象 | 发言者、收文者或受众是否准确 |
| 目的与内容 | 是否完成宣传、倡议、汇报、介绍、动员等任务 |
| 要点 | 是否覆盖材料中要求传达的事实、措施或要求 |
| 语体 | 语气是否符合讲话、倡议、通知、汇报等场景 |
| 格式 | 标题、称谓、正文、落款等要素是否按题干需要保留 |
| 规范 | 字数、条理、称谓和结尾是否合格 |

## 文种处理

- 讲话稿、发言稿：重视对象感、开场、称呼、号召和结束语。
- 倡议书、宣传稿：重视倡议对象、行动要求和动员语气。
- 调研报告、工作汇报：重视事实、问题、做法和条理。
- 通知、请示、函等：可参考 [[来源说明/GitHub开放资料来源]] 中记录的公文规范，但只检查题干要求的格式要素，不把 GB/T 9704 的版头、字号、页码等正式发文规范用于答题纸扣分。

## 批改步骤

1. 从题干抽取身份、对象、目的、文种、材料范围和字数。
2. 判断正文是否服务于沟通目的，而不是写成大作文。
3. 对照材料检查要点覆盖，区分漏写与合理压缩。
4. 检查格式、称谓、语气和结尾是否与文种匹配。
5. 提供一版保留原有身份和任务的局部改写。

## 常见失分

- `应用文/身份错位`：以错误身份发言或称谓不匹配。
- `应用文/任务缺失`：没有完成倡议、说明、汇报或动员任务。
- `应用文/格式不全`：遗漏题干要求的标题、称谓或落款。
- `应用文/语体失当`：语言过于口语化、行政化或泛议论化。
