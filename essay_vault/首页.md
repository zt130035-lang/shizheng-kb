# 申论批改知识库

## 当前目标

让小程序的申论批改结果做到：

- 评分有依据
- 失分点具体
- 修改建议可执行
- 解析能够对应题目材料
- 输出风格稳定统一

## 批改链路

```mermaid
flowchart LR
    A[作文图片] --> B[OCR识别]
    B --> C[识别题型和要求]
    C --> D[检索评分标准]
    D --> E[检索相似批改案例]
    E --> F[检索政策素材]
    F --> G[生成结构化批改结果]
```

## 维护顺序

1. 先完善 [[评分标准/总则]] 和各题型规则。
2. 再整理 [[问题标签/问题标签索引]]。
3. 每种题型录入至少 10 条人工复核案例。
4. 最后补充政策素材和高分表达。
